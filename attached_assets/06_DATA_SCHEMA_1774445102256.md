# DATA SCHEMA — Calcolatore RAL → Netto

Questo documento specifica la struttura dati del calcolatore: input dell'utente, configurazione fiscale, output del calcolo.

---

## 1. INPUT

| Campo | Tipo | Vincoli | Default | Note |
|-------|------|---------|---------|------|
| `ral` | `number` (intero) | min: 1, max: 500000 | — | Obbligatorio. L'unico input utente |

**Validazione:**
- Se vuoto o 0 → errore "Inserisci una RAL valida"
- Se < 1 o > 500.000 → errore "Inserisci una RAL tra €1 e €500.000"
- Caratteri non numerici → filtrati in input (solo cifre accettate)
- Formattazione: separatore migliaia con punto (es. "30.000")

---

## 2. CONFIGURAZIONE FISCALE (COSTANTI)

Tutte le costanti sono raggruppate in un oggetto in cima al codice JS. Aggiornamento annuale = modificare questo blocco.

### 2.1 Contributi INPS

```javascript
const INPS = {
  aliquota_base: 0.0919,           // 9.19% a carico dipendente
  soglia_aggiuntivo: 56224,        // Prima fascia pensionabile 2026 (Circolare INPS 6/2026)
  aliquota_aggiuntivo: 0.01        // 1% su quota eccedente
}
```

### 2.2 Scaglioni IRPEF

```javascript
const IRPEF_SCAGLIONI = [
  { limite: 28000, aliquota: 0.23 },
  { limite: 50000, aliquota: 0.33 },   // 33% dal 2026 (era 35% nel 2025)

```

## 3. OUTPUT

### 3.1 Valori calcolati

| Campo | Tipo | Descrizione |
|-------|------|-------------|
| `contributi_inps_base` | `number` | RAL × 9.19% |
| `contributi_inps_aggiuntivo` | `number` | (RAL - 56224) × 1% se RAL > 56224, altrimenti 0 |
| `contributi_inps_totale` | `number` | Somma dei due |
| `imponibile_irpef` | `number` | RAL - contributi INPS totale |
| `irpef_lorda` | `number` | Calcolata per scaglioni progressivi |
| `detrazione_lavoro_dip` | `number` | Art. 13 TUIR |
| `ulteriore_detrazione_cuneo` | `number` | 0 se fuori fascia €20K-€40K |
| `irpef_netta` | `number` | max(0, lorda - detrazioni) |
| `trattamento_integrativo` | `number` | 0 se reddito > €15K |
| `bonus_cuneo` | `number` | 0 se reddito > €20K |
| `addizionale_regionale` | `number` | Calcolata per scaglioni Lombardia |
| `addizionale_comunale` | `number` | 0 se imponibile ≤ €23K |
| `totale_trattenute` | `number` | INPS + IRPEF netta + addizionali |
| `totale_bonus` | `number` | bonus cuneo + trattamento integrativo |
| `netto_annuale` | `number` | RAL - trattenute + bonus |
| `netto_mensile_13` | `number` | netto annuale ÷ 13 |
| `netto_mensile_12` | `number` | netto annuale ÷ 12 |
| `aliquota_effettiva` | `number` | (trattenute - bonus) / RAL × 100 |

### 3.2 Dati derivati per la UI

| Campo | Usato da | Descrizione |
|-------|----------|-------------|
| Percentuale di ogni voce sulla RAL | Breakdown, barra | `voce / RAL × 100` |
| Flag voci condizionali | Breakdown, note | Quali righe mostrare (bonus, aggiuntivo INPS, esenzione comunale) |
| Lista semplificazioni | Note | Testo dinamico basato sulla fascia di reddito |

## 4. FLUSSO DATI

```
[Input: RAL]
    │
    ▼
[Validazione] ──errore──► [Messaggio errore]
    │
    ok
    │
    ▼
[Calcolo] ← legge CONFIGURAZIONE FISCALE
    │
    ▼
[Output: oggetto con tutti i campi §3.1]
    │
    ├──► [Render Summary Cards]
    ├──► [Render Barra Distribuzione]
    ├──► [Render Breakdown]
    └──► [Render Note condizionali]
```

Nessuna persistenza. Nessun state management. Calcolo puro: `f(RAL, CONFIG) → OUTPUT`.
