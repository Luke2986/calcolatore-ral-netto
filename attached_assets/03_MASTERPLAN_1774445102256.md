## 2. ORDINE DI COSTRUZIONE DEI COMPONENTI FRONTEND

Il frontend ha 4 componenti. L'ordine di costruzione segue le dipendenze.

```
[1] Input + Engine di calcolo
     │
     ├──► [2] Summary cards (netto annuale, mensile, aliquota)
     │
     ├──► [3] Breakdown tabellare (tutte le voci)
     │
     └──► [4] Barra distribuzione (visualizzazione proporzionale)
```

- **[1] è bloccante** per tutto il resto. Se il motore di calcolo è sbagliato, tutto è sbagliato.
- **[2] e [3]** sono indipendenti tra loro ma dipendono da [1].
- **[4]** dipende da [1] ma può essere costruito in parallelo a [2] e [3].
- Le **note/semplificazioni** sono statiche + condizionali (mostrano voci diverse in base alla fascia di reddito). Dipendono da [1] per sapere quali condizioni mostrare.

## 3. DECISIONI ARCHITETTURALI

| Decisione | Scelta | Alternativa scartata | Motivazione |
|-----------|--------|---------------------|-------------|
| File singolo vs multi-file | File HTML singolo | React app con componenti | Il prototipo deve essere apribile con un doppio click, senza build step. L'obiettivo è dimostrare capacità, non architettura scalabile |
| Costanti fiscali inline vs file config | Oggetto JS in cima al file | JSON separato | Un file singolo significa zero dipendenze. Le costanti sono raggruppate e commentate per aggiornamento facile |
| CSS custom vs framework | CSS custom con variabili | Tailwind | Controllo totale sul design, nessuna dipendenza CDN, file più leggero |
| Font da CDN vs system | Google Fonts (DM Sans + DM Mono) | System fonts | Il design system specifica font precisi. Il CDN è accettabile per un prototipo |
| Calcolo client-side vs server | Client-side puro | API backend | Nessun dato sensibile, nessun motivo per un server. Funziona offline |
| Formattazione numeri | `toLocaleString('it-IT')` | Libreria esterna | Nativa, corretta per il locale italiano, zero dipendenze |

## 4. DIPENDENZE CRITICHE E MITIGAZIONE

| Dipendenza | Rischio | Probabilità | Mitigazione |
|-----------|---------|-------------|-------------|
| Correttezza delle formule fiscali 2026 | Le formule nel RESEARCH_LOG potrebbero avere errori | Media | Verifica incrociata con 2-3 fonti per ogni formula. Test su casi noti (RAL 25K, 30K, 40K, 50K) confrontati con calcolatori online |
| Addizionali regionali Lombardia | Le aliquote potrebbero essere cambiate o applicate con metodo diverso | Bassa | Fonte: sito Regione Lombardia + Coverflex. Il prototipo dichiara la fonte |
| Bonus cuneo fiscale 2025 | Logica nuova (primo anno), possibili interpretazioni diverse | Media | Fonte: Circolare AdE 4/2025, NoiPA. Il prototipo implementa il caso semplice (solo reddito da lavoro dipendente) |
| Google Fonts non carica | CDN down o bloccato | Bassa | Fallback su system font nel CSS (`-apple-system, sans-serif`) |

## 5. CRITERI DI FINE (DEFINITION OF DONE)

Il prototipo è "fatto" quando:

- [ ] **Calcolo corretto:** RAL €30.000 → netto coerente con PMI.it/QuantoPrendo (delta ≤ 2%)
- [ ] **Calcolo corretto:** RAL €50.000 → netto coerente (delta ≤ 2%)
- [ ] **Calcolo corretto:** RAL €20.000 → bonus cuneo applicato, voce visibile nel breakdown
- [ ] **Calcolo corretto:** RAL €8.000 → no-tax area gestita, IRPEF netta = 0
- [ ] **Breakdown completo:** tutte le voci della pipeline (INPS, IRPEF lorda, detrazioni, IRPEF netta, addizionali, bonus)
- [ ] **Note visibili:** semplificazioni dichiarate nella UI
- [ ] **Responsive:** testato su viewport 375px (mobile)
- [ ] **Codice leggibile:** funzioni nominate, costanti raggruppate, commenti sulle formule
- [ ] **Documentazione completa:** tutti i documenti coerenti e con riferimenti incrociati
- [ ] **README** con ordine di lettura e stato del progetto

## 6. COSA VIENE DOPO IL PROTOTIPO (FUORI SCOPE)

Se il prototipo viene validato, le evoluzioni naturali sarebbero:

1. **Parametri configurabili:** regione, comune, tipo contratto, CCNL, familiari a carico
2. **Confronto tra RAL:** input di 2 RAL side-by-side per simulare scenari
3. **Integrazione in JetHR:** modale dentro il flusso di creazione offerta
4. **Esportazione PDF:** scheda riassuntiva da allegare all'offerta
5. **Aggiornamento multi-anno:** switch anno fiscale per confronto 2025 vs 2026

Nessuna di queste è in scope per il prototipo.
